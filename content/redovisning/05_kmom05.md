Redovisning kmom05
=========================

Att använda Google´s PageSpeed Insights var nytt. Nätverksfliken i Devtools var däremot bekant sedan tidigare.
Visserligen har jag inte haft någon anledning att analysera hastigheten på någon webbplats tidigare, men det var intressant att se hur det kan göras.

Som analysen visar så används ofta tyngre bilder än nödvändigt. Jag uppfattar det som att dagens moderna användare "kräver" att det finns mycket bilder på sidorna.
Vi är vana vid att presenteras information grafiskt och har inte tid att läsa mer än nödvändigt. Man brukar ju säga att "en bild säger mer än tusen ord",
vilket jag håller med om.

Att behandla bilder med Cimage känns väldigt bra. Jag visste ärligt talat inte att det fanns denna typ av verktyg, utan var i tron att själva bilden måste anpassas
innan "uppladdning" och att de möjligheter som fanns att anpassa den för sin webbsida var med enbart CSS.
Det är skönt att inte behöva modifiera själva bilden kan jag tycka, utan att istället kunna använda
samma bild på flera olika sätt. Dessutom kan man lätt anpassa bilden och få ner vikten ordentligt.

Modulen @desinax/figure tycker jag om att arbeta med även den. Att lägga in bilder med shortcode FIGURE och lätt anpassa dem till att flyta höger, vänster eller vara centrerade
går snabbt och bildtexten hamnar precis där den ska.

Att anpassa webbsidor efter mobila enheter förstår jag vikten av bara genom att se till mitt eget användande.
Nittio procent av mitt surfande sker från mobil, om man bortser från då jag använder datorn i arbete eller studier.

TIL för detta kmom är att man kan anpassa bilder direkt vid inläsning istället för att processa dem i förväg i bildbehandlingsprogram.
